# Компоненты

Описание всех React-компонентов фронтенда.

## App.tsx

Корневой компонент. Инициирует загрузку тикетов и приоритетов при монтировании; тикеты рефетчатся при изменении фильтров.

**Redux:** `useAppDispatch`, `useAppSelector` (loading, error, filters, priorities)

**Структура:**
```
<header> — заголовок + LoginModal
<main>   — TicketForm + TicketFilters + ErrorMessage + TicketTable + Pagination
```

---

## LoginModal

Модальное окно входа/выхода администратора.

**Redux:** `loginAsync`, `logout`, `clearError`, `selectIsAdmin`, `selectAuthLoading`, `selectAuthError`

**Поведение:**
- Кнопка "Вход администратора" / "Выйти" в хедере
- При клике открывается модальное окно с формой
- Токен сохраняется в `sessionStorage`
- При ошибке — отображение сообщения от бэкенда

---

## TicketForm

Форма создания нового тикета.

**Redux:** `createTicket`, `selectTicketsLoading`, `selectPriorities`

**Поля:**
| Поле | Валидация |
|------|-----------|
| Название | 3–120 символов |
| Приоритет | select из динамического списка приоритетов (из `/api/priorities`) |
| Описание | До 1000 символов (необязательно) |

**Поведение:**
- Клиентская валидация перед отправкой
- После успешного создания — сброс формы
- Ошибки отображаются под полями

---

## TicketFilters

Фильтры, поиск и сортировка.

**Redux:** `setFilters`, `selectFilters`, `selectPriorities`

**Элементы:**
- Поиск по названию/описанию (debounce 300мс)
- Фильтр по статусу
- Фильтр по приоритету (динамически из `/api/priorities`)
- Сортировка (дата/приоритет, asc/desc)

**Поведение:**
- При изменении любого фильтра — сброс страницы на 1
- Автоматический рефетч через `useEffect` в App

---

## TicketTable

Таблица со списком тикетов.

**Redux:** `selectTickets`, `selectIsAdmin`, `updateTicketStatus`, `deleteTicket`

**Колонки:**
| Колонка | Содержимое |
|---------|-----------|
| Название | `ticket.title` |
| Описание | `ticket.description` (или "-") |
| Статус | `<select>` с валидными переходами |
| Приоритет | Бейдж с `ticket.priority_name` |
| Создан | Дата в локальном формате |
| Изменён | Дата в локальном формате |
| Действия | Кнопка удаления (только для admin, кроме done) |

**Валидные переходы статусов:**
- `new` → `new`, `in_progress`, `done`
- `in_progress` → `new`, `in_progress`, `done`
- `done` → заблокирован

**Оптимизация:** `React.memo` на строках таблицы (`TicketRow`).

---

## Pagination

Навигация по страницам.

**Redux:** `setPage`, `selectPagination`

**Поведение:**
- Окно из 7 страниц (с эллипсисом "...")
- Кнопки "Назад" / "Далее"
- Скрывается при ≤1 странице

---

## ErrorMessage

Баннер ошибки с кнопкой закрытия.

**Пропсы:**
| Пропс | Тип | Описание |
|-------|-----|----------|
| `message` | string | Текст ошибки |
| `onDismiss` | () => void | Колбэк закрытия (необязательный) |

**Поведение:**
- Розовый фон, красный текст
- Кнопка "Закрыть" (только если передан `onDismiss`)
