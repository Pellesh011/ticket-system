# API

REST API для управления тикетами.

## Базовый URL

```
http://localhost:8000/api
```

## Эндпоинты

### Health Check

```
GET /api/health
```

**Ответ (200):**
```json
{ "status": "ok" }
```

---

### Авторизация

#### Вход

```
POST /api/auth/login
```

**Тело запроса:**
```json
{
  "username": "admin",
  "password": "admin"
}
```

**Ответ (200):**
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer"
}
```

**Ошибки:**
- `401` — Неверные учётные данные
- `429` — Превышен лимит запросов (10/мин)

#### Верификация токена

```
GET /api/auth/verify
```

**Заголовки:**
```
Authorization: Bearer <token>
```

**Ответ (200):**
```json
{ "valid": true }
```

**Ошибки:**
- `401` — Требуется заголовок авторизации
- `401` — Неверный заголовок авторизации
- `401` — Неверный или истёкший токен

### Тикеты

#### Создание

```
POST /api/tickets
```

**Тело запроса:**
```json
{
  "title": "Название тикета",
  "description": "Описание (необязательно)",
  "priority": "normal"
}
```

**Поля:**
| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `title` | string | да | 3–120 символов |
| `description` | string | нет | До 1000 символов |
| `priority` | string | нет | `low`, `normal` (по умолчанию), `high` |

**Ответ (201):**
```json
{
  "id": 1,
  "title": "Название тикета",
  "description": "Описание",
  "status": "new",
  "priority": "normal",
  "created_at": "2026-06-28T12:00:00Z",
  "updated_at": "2026-06-28T12:00:00Z"
}
```

---

#### Список (с фильтрами, сортировкой, пагинацией)

```
GET /api/tickets
```

**Параметры запроса:**
| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `status` | string | — | Фильтр по статусу: `new`, `in_progress`, `done` |
| `priority` | string | — | Фильтр по приоритету: `low`, `normal`, `high` |
| `search` | string | — | Поиск по названию и описанию |
| `sort_by` | string | `created_at` | Сортировка: `created_at`, `priority` |
| `sort_order` | string | `desc` | Направление: `asc`, `desc` |
| `page` | int | `1` | Номер страницы (≥1) |
| `page_size` | int | `20` | Элементов на странице (1–100) |

**Пример:**
```
GET /api/tickets?status=new&priority=high&sort_by=created_at&sort_order=desc&page=1&page_size=10
```

**Ответ (200):**
```json
{
  "items": [
    {
      "id": 1,
      "title": "Тикет",
      "description": null,
      "status": "new",
      "priority": "high",
      "created_at": "2026-06-28T12:00:00Z",
      "updated_at": "2026-06-28T12:00:00Z"
    }
  ],
  "total": 42,
  "page": 1,
  "page_size": 10,
  "total_pages": 5
}
```

---

#### Получение по ID

```
GET /api/tickets/{id}
```

**Ответ (200):** Тикет (см. выше)

**Ошибки:**
- `404` — Тикет с id {id} не найден

---

#### Обновление

```
PATCH /api/tickets/{id}
```

**Тело запроса:**
```json
{
  "title": "Новое название",
  "description": "Новое описание",
  "priority": "high"
}
```

Все поля необязательные. Обновляются только переданные.

**Ответ (200):** Обновлённый тикет

**Ошибки:**
- `400` — Невозможно edit тикет в статусе 'выполнено'
- `404` — Тикет с id {id} не найден

---

#### Изменение статуса

```
PATCH /api/tickets/{id}/status
```

**Тело запроса:**
```json
{
  "status": "in_progress"
}
```

**Валидные переходы:**
| Текущий статус | Доступные переходы |
|----------------|-------------------|
| `new` | `new`, `in_progress` |
| `in_progress` | `new`, `in_progress`, `done` |
| `done` | — (блокируется) |

**Ответ (200):** Обновлённый тикет

**Ошибки:**
- `400` — Невозможно перейти из '{current}' в '{target}'
- `400` — Невозможно change status of тикет в статусе 'выполнено'
- `404` — Тикет с id {id} не найден

---

#### Удаление

```
DELETE /api/tickets/{id}
```

**Заголовки (требуется для admin):**
```
Authorization: Bearer <token>
```

**Ответ (204):** Нет тела

**Ошибки:**
- `400` — Невозможно delete тикет в статусе 'выполнено'
- `403` — Требуется доступ администратора
- `404` — Тикет с id {id} не найден

---

## Коды ответов

| Код | Описание |
|-----|----------|
| `200` | Успешный запрос |
| `201` | Ресурс создан |
| `204` | Успешное удаление (нет тела) |
| `400` | Ошибка бизнес-правил |
| `401` | Не авторизован |
| `403` | Нет доступа |
| `404` | Ресурс не найден |
| `422` | Ошибка валидации (Pydantic) |
| `429` | Превышен лимит запросов |
